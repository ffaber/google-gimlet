Class Overview

This class provides a testing-based implementation of NestedScope. It has identical functionality, but it doesn't care about enter(ScopeId), enterNew(), or exit(), because these methods are all no-ops. As soon as an instance of this class is created, scope is entered for the creating thread.

public void enter (ScopeId scopeId)

This method enters into a new nested scope. It may be called during a unit of work, or at the start of a unit of work. It is preferable to call enterNew() or enterNew(ScopeId) at the beginning of a new unit of work, because those methods will check whether the current thread is already within a unit of work. Hence, this method must be used to enter a nested unit of work.

If the given ScopeId is a duplicate of an outer scope, then an IllegalArgumentException is thrown.
